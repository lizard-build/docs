# Concepts

The core model behind Lizard: how your code is organized, how it's built, and how a release reaches production. Read these once and the rest of the docs will make sense.

## In this section

- [Architecture](/concepts/architecture) — the workspace → project → service hierarchy, managed addons, and cross-resource references.
- [Build Pipeline](/concepts/build-pipeline) — how source becomes a container image (synthesized Dockerfile, repo Dockerfile, or lizardpack auto-detect) and what triggers a rebuild.
- [Deployments](/concepts/deployments) — the build-and-release lifecycle, restarts vs. redeploys, and live config changes.
