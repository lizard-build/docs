# Deploy

Everything about getting a service running on Lizard and keeping it that way — from your first push to scaling and region placement.

## In this section

- [Deploy from GitHub](/deploy/github) — connect a repo so pushes to the tracked branch auto-redeploy.
- [Deploy from Local Code](/deploy/upload) — ship the current directory with `lizard up`, no Git required.
- [Background Workers](/deploy/workers) — run queue consumers and polling loops with `containerPort=0`.
- [Scaling](/deploy/scaling) — adjust replicas, resources, and storage.
- [Regions](/deploy/regions) — where your services run.
