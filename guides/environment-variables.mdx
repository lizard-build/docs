# Environment Variables

Manage secrets and configuration across environments.

## Local Development

Create a `.env.local` file in your project root:

```bash
LIZARD_API_KEY=lzd_dev_abc123
DATABASE_URL=postgresql://localhost:5432/mydb
REDIS_URL=redis://localhost:6379
```

This file is automatically added to `.gitignore` by `lizard init`.

## Setting Variables

Use the CLI to set environment variables for your deployed app:

```bash
lizard env set DATABASE_URL="postgresql://prod-host:5432/mydb"
lizard env set REDIS_URL="redis://prod-cache:6379"
```

## Listing Variables

```bash
lizard env list
```

Output:

```
┌─────────────────┬────────────────┬─────────────┐
│ Name            │ Value          │ Environment │
├─────────────────┼────────────────┼─────────────┤
│ DATABASE_URL    │ ••••••••••     │ production  │
│ REDIS_URL       │ ••••••••••     │ production  │
│ LIZARD_API_KEY  │ ••••••••••     │ all         │
└─────────────────┴────────────────┴─────────────┘
```

## Per-Environment Variables

Set different values for different environments:

```bash
lizard env set DATABASE_URL="postgresql://staging:5432/mydb" --env staging
lizard env set DATABASE_URL="postgresql://prod:5432/mydb" --env production
```

## Secrets

For sensitive values, use `lizard secret`:

```bash
lizard secret set STRIPE_KEY="sk_live_..."
```

Secrets are encrypted at rest and only decrypted at runtime.
